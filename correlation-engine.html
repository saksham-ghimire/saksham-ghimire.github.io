Correlation Engine

  1. Projects
  2. Correlation Engine

Description

The correlation engine developed in Golang serves as a robust solution for detecting anomalies in network logs, embodying a powerful Pub/Sub architecture implemented through Golang channels. This engine is meticulously designed to analyze vast sets of data and identify potential security threats based on predefined rules set by Security Operations Center (SOC) analysts.

At its core, the engine utilizes Golang channels to facilitate seamless communication between various components, allowing for efficient data flow within the system. This architecture ensures a streamlined process of real-time correlation, enhancing the engine's responsiveness to potential security incidents.

The primary objective of the correlation engine is to generate near-real-time alerts when deviations from established norms are detected. These alerts are then communicated promptly to designated parties through multiple channels, such as email, Rocket.Chat, Slack, or any preferred communication channel. This flexibility in notification mechanisms ensures that relevant stakeholders receive timely information, enabling swift response and mitigation measures.

By combining the efficiency of Golang, a Pub/Sub architecture, and rule-based anomaly detection, the correlation engine stands as a sophisticated tool in the arsenal of cybersecurity, providing proactive threat identification and response capabilities for a secure and resilient network infrastructure.

Stacks Used

  • Golang: Using channel based pub/sub approach to match rule against high volume of logs through multiple sources
  • Kafka: To push alert event to respective channels which delivered the message to designated party
  • Elasticsearch: Pushing alert over to elasticsearch for log keeping and visualization purposes
  • GRPC: Providing interface/api to communicate rule status to correlation engine
