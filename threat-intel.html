Threat Intel

  1. Projects
  2. Threat Intel

Description

The Threat Intelligence project is a dynamic cybersecurity solution leveraging Golang and Python to enhance threat detection and analysis. Primarily written in Golang, the system efficiently processes a high volume of logs ingested from Logstash, utilizing buffered channels for optimized queuing. The core functionality includes concurrent querying of IP addresses against open-source platforms like MISP, generating alerts promptly pushed to Elasticsearch. To broaden its intelligence scope, Python is employed to crawl open-source websites such as blocklist.de and ellio.tech. Periodically, the program retrieves data from these sources, enriching its threat intelligence database.

The versatility of Python in web crawling ensures adaptability to diverse data sources, contributing to a comprehensive understanding of potential threats. The collected information from open-source websites is seamlessly stored in Elasticsearch and Redis, enhancing the project's analytical capabilities. Redis serves as a quick caching mechanism, optimizing performance by reducing repetitive queries.

In summary, the Threat Intelligence project combines the strengths of Golang and Python, offering a scalable, efficient, and versatile solution. It not only processes logs and queries open-source platforms but also proactively gathers and stores threat intelligence data from online sources, making it a robust tool for real-time threat detection in the ever-evolving cybersecurity landscape.

Stacks Used

  • Golang: Concurrently querying MISP and other Opensource threat intelligence to verify log, featured buffered channel queuing system for efficiency
  • Elasticsearch: Pushing alert over to elasticsearch for log keeping and visualization purposes
  • Logstash: Responsible for forwarding high volume of logs to program through UDP port
  • Redis: Used as a quick caching solution to reduce query overhead to multiple sources
  • Python: Used to crawl data from multiple sources and feed into threat-intel database
